## 优先级：高

### 1. PWM输入周期捕获可能跨周期 (pwm_input.c)

**位置**：`pwm_input.c:15-34`

**问题描述**：
```c
pwmin.period = (PWMB_CCR1H << 8) + PWMB_CCR1L;  // CC1捕获周期
pwmin.high_value = (PWMB_CCR2H << 8) + PWMB_CCR2L;  // CC2捕获高电平
```

当CC1捕获中断和CC2捕获中断发生时，pwmin.period和pwmin.high_value可能来自**不同的PWM周期**。

**影响**：可能导致油门信号被错误判断为无效，电机运行不稳定。

**状态**：暂未修复

---

### 2. 闭环运行时占空比调整过慢 (motor.c)

**位置**：`motor.c:630-644`

**问题描述**：
```c
if(0 == (motor.commutation_num%6))  // 每6次换相才调整1
{
    if(motor.duty > motor.duty_register)
    {
        motor.duty_register += 1;  // 仅增加1
    }
}
```

闭环运行阶段（run_flag=4）占空比每6次换相才能调整1。对于高速电机，响应会非常迟缓。

**影响**：转速响应迟缓，调速不跟手。

**建议**：可以根据电机转速动态调整增量。

**状态**：暂未修复

---

### 3. S曲线加速使用浮点exp() (motor.c)

**位置**：`motor.c:56`

**问题描述**：
```c
delay = START_OPEN_LOOP_MIN_DELAY + ... / (1.0f + exp((ratio - 0.5f) * 10));
```

STC32G是8位机，没有FPU，`exp()`浮点运算非常耗时，可能导致开环换相定时不准确。

**影响**：开环启动可能不稳定，定时精度差。

**建议**：改用查表法或近似多项式。

**状态**：暂未修复

## 优先级：中

### 4. 定时器3重配置系数近似 (motor.c)

**位置**：`motor.c:268`

**问题描述**：
```c
us = us*5/2;  // 注释说基于30M，但实际是33.1776MHz
```

系数5/2=2.5是基于30MHz计算的近似值，实际33.1776MHz下有偏差。对于START_OPEN_LOOP_MAX_DELAY=3500us：
- 期望：3500us
- 实际：约3166us（误差约10%）

**影响**：启动加速曲线与预期有偏差。

**状态**：暂未修复

### 5. 比较器初始化未使能中断 (comparator.c)

**位置**：`comparator.c:100-108`

**问题描述**：
```c
void comparator_init(void)
{
    CMPCR1 = 0x8C;  // 只配置了比较器，未使能中断
    CMPCR2 = 63;
}
```

比较器中断在motor_start()时才通过设置比较器边沿触发来使能。初始化函数名有误导性。

**影响**：无严重影响，这是设计如此。

**状态**：暂未修复（设计如此）
